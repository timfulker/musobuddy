Pre-Deployment Checklist

Test Instructions

  1. Start the dev server: npm run dev
  2. Open Chrome DevTools (F12)
  3. Toggle device emulation (Ctrl+Shift+M)
  4. Select iPhone 12 Pro from device list
  5. Navigate to: http://localhost:5173/client-portal/test?token=test123
  6. Check the following:

What to Check in DevTools

// 1. Check if body has the correct classes
document.body.classList.contains('client-portal-active')  // Should be true
document.body.classList.contains('no-mobile-nav')         // Should be true

// 2. Check if mobile nav element exists
document.querySelector('[data-mobile-nav]')               // Should be null or hidden
document.querySelector('#mobile-nav-main')                // Should be null or hidden
document.querySelector('.mobile-nav-component')           // Should be null or hidden

// 3. Check for any visible fixed bottom elements
Array.from(document.querySelectorAll('.fixed.bottom-0')).filter(el => {
    const styles = window.getComputedStyle(el);
    return styles.display !== 'none' && !el.classList.contains('client-portal-allowed');
})  // Should return empty array []

// 4. Check if MutationObserver is active
// Look in console for any elements with attribute: data-hidden-by-client-portal="true"

// 5. Check computed styles on navigation
const nav = document.querySelector('[data-mobile-nav]');
if (nav) {
    const styles = window.getComputedStyle(nav);
    console.log({
        display: styles.display,        // Should be 'none'
        visibility: styles.visibility,  // Should be 'hidden'
        position: styles.position,      // Should be 'absolute' or 'fixed'
        left: styles.left,             // Should be '-99999px' or similar
        pointerEvents: styles.pointerEvents  // Should be 'none'
    });
}

Edge Cases to Test

  • Navigate directly to client portal URL (not from another page)
  • Navigate from dashboard to client portal
  • Refresh the page while on client portal
  • Check in both portrait and landscape orientation
  • Test with different mobile device emulations (iPhone SE, Pixel 5)
  • Test with slow network throttling (to catch lazy-loaded elements)
